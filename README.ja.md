> # TBM — The Bitcoin Machine (Umbrel 1.x 互換フォーク)
> 
> > **これは非公式のコミュニティフォークです。** これは、オリジナルの [doidotech/TBM](https://github.com/doidotech/TBM) プロジェクトを、**Umbrel OS 1.x** および **Pillow 10+** 環境で動作するように修正したバージョンです。
> > オリジナルプロジェクトはもはやメンテナンスされておらず、このフォークは多くのユーザーがUmbrelのアップグレード後に経験する **LCDホワイトスクリーン（フリーズ）問題** を解決します。
> 
> ---
> 
> ## 初心者向けインストールガイド（Macユーザー向け）
> 
> このガイドは、ターミナルやコーディングの経験がない方でも簡単に従えるように書かれています。手順を注意深く進めてください。
> 
> ### ステップ1：Umbrelノードにリモートで接続する（SSH）
> 
> まず、お使いのMacコンピュータからUmbrelノード（Raspberry Pi）にリモートで接続する必要があります。これには「ターミナル」アプリを使用します。
> 
> 1.  **ターミナルアプリを開く**
>     *   `Command (⌘)` + `Space` キーを押してSpotlight検索を開き、`Terminal`と入力してEnterキーを押します。
> 
> 2.  **SSHコマンドで接続する**
>     *   ターミナルウィンドウに以下のコマンドを入力します。`umbrel.local`は、ほとんどのホームネットワークで機能するデフォルトのアドレスです。
> 
>     ```bash
>     ssh umbrel@umbrel.local
>     ```
> 
> 3.  **パスワードを入力する**
>     *   パスワードの入力を求められます。これは、Umbrelダッシュボードにログインする際に使用するパスワードと同じです。
>     *   **注意：** セキュリティのため、パスワードを入力しても画面には何も表示されません。そのまま入力してEnterキーを押してください。
> 
>     ```bash
>     umbrel@umbrel.local's password:
>     ```
> 
> 4.  **接続成功の確認**
>     *   接続に成功すると、Umbrelのロゴとともにウェルカムメッセージが表示されます。これ以降、入力するすべてのコマンドはUmbrelノードで実行されます。
> 
>     ```
>        _   _ ____  _   _ ____  _     
>       | | | | __ )| | | | __ )| |    
>       | | | |  _ \| | | |  _ \| |    
>       | |_| | |_) | |_| | |_) | |___ 
>        \___/|____/ \___/|____/|_____|
>     ```
> 
> ### ステップ2：コードをダウンロードしてインストールスクリプトを実行する
> 
> 次に、ホワイトスクリーン問題を解決したコードをダウンロードしてインストールします。以下のコマンドを**一行ずつコピーしてターミナルに貼り付け、Enterキーを押してください。**
> 
> 1.  **GitHubからコードをクローンする**
>     *   このコマンドは、私のGitHubリポジトリから修正されたTBMコードをUmbrelノードにダウンロードします。
> 
>     ```bash
>     git clone https://github.com/ghmy4ff4t8-coder/TBM.git
>     ```
> 
> 2.  **作業ディレクトリに移動する**
> 
>     ```bash
>     cd TBM/TBMLCD-v0.5/UmbrelLCDV2_0
>     ```
> 
> 3.  **セットアップスクリプトを実行する**
>     *   このスクリプトは、LCDが動作するために必要なすべてのプログラムとライブラリを自動的にインストールします。
> 
>     ```bash
>     chmod +x lcdSetupScript.sh
>     ./lcdSetupScript.sh
>     ```
> 
>     *   インストール中に多くのテキスト行がスクロール表示されます。`Setup complete!`というメッセージが表示されれば成功です。
> 
> ### ステップ3：Umbrelノードを再起動する
> 
> すべての設定を正しく適用するために、システムを再起動する必要があります。
> 
> ```bash
> sudo reboot
> ```
> 
> *   このコマンドを実行すると、SSHセッションが切断されます。Umbrelノードが完全に再起動するまで、約3〜5分お待ちください。
> 
> ### ステップ4：LCDサービスをセットアップして開始する
> 
> 再起動が完了したら、ステップ1の方法で再度UmbrelノードにSSHで接続し、以下のコマンドを順番に入力します。
> 
> 1.  **再度作業ディレクトリに移動する**
> 
>     ```bash
>     cd ~/TBM/TBMLCD-v0.5/UmbrelLCDV2_0
>     ```
> 
> 2.  **サービスセットアップスクリプトを実行する**
> 
>     ```bash
>     chmod +x umbrelLCDServiceSetup.sh
>     ./umbrelLCDServiceSetup.sh
>     ```
> 
> 3.  **画面と通貨を選択する（非常に重要）**
>     *   スクリプトを実行すると、LCDに表示する画面と、ビットコインの価格を表示する通貨（USD、EUR、KRWなど）を尋ねられます。
>     *   各質問に`yes`または`no`で答えてEnterキーを押します。
>     *   最後に、希望する通貨コード（例：`USD`）を入力すると、すべての設定が完了し、LCDサービスが自動的に開始されます。
> 
> これで、TBM LCDが正常に動作するのを確認できます！
>

> ## トラブルシューティング
> 
> **LCDがまだ白い画面のままです：**
> *   まず、配線が正しいことを確認してください（以下の配線図を参照）。
> *   `./lcdSetupScript.sh`と`./umbrelLCDServiceSetup.sh`を再度実行してみてください。
> 
> **サービスが正しく実行されているか確認したい：**
> *   以下のコマンドでサービスのステータスを確認できます：
>     ```bash
>     sudo systemctl status UmbrelST7735LCD
>     ```
> *   リアルタイムのログを表示するには、以下のコマンドを入力します。エラーメッセージを確認するのに役立ちます。
>     ```bash
>     sudo journalctl -u UmbrelST7735LCD -f
>     ```

> 
> | LCDピン | Raspberry Piピン番号 | GPIO | 説明 |
> | :--- | :--- | :--- | :--- |
> | VCC | Pin 1 | 3.3V | 電源 |
> | GND | Pin 6 | GND | グラウンド |
> | SCL/CLK | Pin 23 | GPIO 11 | SPIクロック |
> | SDA/MOSI | Pin 19 | GPIO 10 | SPIデータ |
> | RES/RST | Pin 22 | GPIO 25 | リセット |
> | DC | Pin 18 | GPIO 24 | データ/コマンド選択 |
> | CS | Pin 24 | GPIO 8 | チップセレクト |
> | BL/LED | Pin 17 | 3.3V | バックライト電源 |

> ## 修正された技術的な問題
> 
> | 問題 | 原因 | 解決策 |
> | :--- | :--- | :--- |
> | **LCDホワイトスクリーン** | Pillow 10.0.0ライブラリで`draw.textsize()`関数が削除された。 | `draw.textbbox()`を使用する互換性関数に置き換えて解決。 |
> | **pipインストール失敗** | Python 3.11+環境でのシステムパッケージ保護ポリシーの強化。 | `pip`コマンドに`--break-system-packages`フラグを追加して解決。 |
> | **サービス実行エラー** | systemdサービスがDockerのパスを認識しない。 | サービスユニットファイルに`PATH`環境変数を明示的に追加。 |
> | **Umbrel 1.x互換性** | Dockerコンテナ名、`bitcoin-cli`および`lncli`の実行方法の変更。 | 複数のコンテナ名を試行し、直接HTTP RPC呼び出しを追加するなど、フォールバックロジックを強化。 |
